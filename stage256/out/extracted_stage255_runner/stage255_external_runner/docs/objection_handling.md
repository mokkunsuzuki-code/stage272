# Objection Handling

## "This does not seem novel enough."
Correct: the goal is not to claim a new cryptographic primitive or proof.
The contribution is a reproducible structure for claim-to-evidence verification.

## "How is this different from existing formal verification?"
This does not replace formal verification.
It complements formal methods by improving reproducibility and evidence traceability.

## "Why include QKD at all?"
QKD is treated as an optional entropy source, not as an unconditional security claim.
This avoids overclaiming while still allowing protocol-level discussion.

## "If it is not production-ready, what is the value?"
Its value is as a research prototype that improves independent reviewability, explicit assumptions, and tamper-evident evidence binding.

## "Could this be applied beyond QKD?"
Yes.
The framework is protocol-agnostic in structure and can be adapted to other security-sensitive systems.
