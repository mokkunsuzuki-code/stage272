# Reviewer Scope

## Objective

This document defines the bounded scope of Stage244 external review.

The purpose of the scope is to make review realistic, lightweight, and honest.

---

## In Scope

The recommended Stage244 review scope includes:

1. Repository README
2. Reviewer quickstart documentation
3. Review verdict level definitions
4. Generated review request packet
5. Evidence and transparency references listed in the packet
6. Review record structure
7. Verification scripts:
   - tools/generate_review_request.py
   - tools/verify_review_record.py

---

## Optional In Scope

Depending on reviewer availability, the reviewer may also inspect:

- selected CI evidence artifacts
- transparency checkpoint files
- claim summary documents
- onboarding-related workflow materials

---

## Out of Scope

The following are out of scope unless explicitly stated otherwise:

- full cryptographic correctness proof
- full repository audit
- full operational deployment review
- legal compliance review
- infrastructure security assessment
- organizational governance audit

---

## Review Principle

A reviewer should only review what is explicitly listed in the packet.

No implicit expansion of scope should be assumed.
