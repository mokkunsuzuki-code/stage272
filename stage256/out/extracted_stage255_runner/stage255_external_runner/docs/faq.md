# FAQ

## Is this a new security proof?
No. This is a reproducible verification framework.

## Does this claim QKD security?
No. QKD is treated as an optional entropy source.

## What is the main contribution?
Making security claims:
- explicit
- reproducible
- verifiable

## Is this production-ready?
No. This is a research prototype.

## Why Merkle proofs?
To bind evidence in a tamper-evident way.

## What problem does this solve?
It reduces ambiguity between:
claim → implementation → evidence

## How is this different from existing protocols?
This focuses on verification structure, not protocol novelty.
